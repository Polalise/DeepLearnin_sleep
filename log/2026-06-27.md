### 18. Issue found and fixed

- Initial validation found duplicate `participant_object_id + calendar_date` rows in:
  - `fitbit_calories_daily.csv`
- Cause:
  - US-style Fitbit calorie timestamps such as `05/24/21 00:00:00` were grouped using a 10-character prefix, producing groups like `05/24/21 0`.
  - After normalization, multiple raw groups collapsed to the same calendar date.
- Fix:
  - Updated `scripts/06_aggregate_daily_features.py` so large MongoDB aggregation accepts a `date_prefix_bytes` parameter.
  - Used `date_prefix_bytes=8` for `calories`.
  - Regenerated daily aggregate files and validation report.
